【GetRebirthInfo函数对比】

Example版本：
-------------
function UGCPlayerState:GetRebirthInfo()
    local rebirthCount = self.UGCPlayerRebirthCount or self.GameData.PlayerRebirthCount or 0
    local requiredLevel = self:GetRebirthRequiredLevel()
    local requiredCombatPower = self:GetRebirthRequiredCombatPower()
    
    local currentLevel = self.GameData.PlayerLevel or 1
    local Player = UGCGameSystem.GetPlayerPawnByPlayerState(self)
    if Player then
        currentLevel = math.floor(UGCAttributeSystem.GetGameAttributeValue(Player, 'Level') or currentLevel)
    end
    
    local currentCombatPower = self:GetCombatPower()
    local canRebirth, reason = self:CanRebirth()
    
    return {
        rebirthCount = rebirthCount,
        requiredLevel = requiredLevel,
        currentLevel = currentLevel,
        requiredCombatPower = requiredCombatPower,
        currentCombatPower = currentCombatPower,
        canRebirth = canRebirth,
        reason = reason,
        maxRebirthCount = #UGCPlayerState.RebirthLevels
    }
end

返回数据说明：
==============
GetRebirthInfo函数返回一个包含以下字段的表：

1. rebirthCount (number)
   - 当前转生次数
   - 优先使用复制属性UGCPlayerRebirthCount
   - 如果没有则使用GameData.PlayerRebirthCount
   - 默认值为0

2. requiredLevel (number)
   - 下次转生需要的等级
   - 从RebirthLevels数组中获取
   - 例如：第1次转生需要25级

3. currentLevel (number)
   - 当前玩家等级
   - 优先从Pawn的Level属性读取
   - 如果没有Pawn则使用GameData.PlayerLevel
   - 默认值为1

4. requiredCombatPower (number)
   - 下次转生需要的战斗力
   - 从RebirthCombatPowers数组中获取
   - 例如：第1次转生需要2000战斗力

5. currentCombatPower (number)
   - 当前玩家战斗力
   - 通过GetCombatPower()计算
   - 公式：MaxHp*5% + Attack*70% + MaxMagic*25%

6. canRebirth (boolean)
   - 是否可以转生
   - 通过CanRebirth()函数判断
   - 需要同时满足等级和战斗力要求

7. reason (string)
   - 转生状态说明
   - 如果可以转生："可以转生"
   - 如果不能转生：说明原因（等级不足/战斗力不足/已达最大次数）

8. maxRebirthCount (number)
   - 最大转生次数
   - 等于RebirthLevels数组的长度
   - 当前设置为7次

UI使用示例：

```lua
local playerState = UGCGameSystem.GetLocalPlayerState()
if playerState and playerState.GetRebirthInfo then
    local info = playerState:GetRebirthInfo()
    
    -- 显示转生次数
    print("已转生次数: " .. info.rebirthCount)
    
    -- 显示等级要求
    print("需要等级: " .. info.requiredLevel)
    print("当前等级: " .. info.currentLevel)
    
    -- 显示战斗力要求
    print("需要战斗力: " .. info.requiredCombatPower)
    print("当前战斗力: " .. info.currentCombatPower)
    
    -- 判断是否可以转生
    if info.canRebirth then
        print("可以转生！")
    else
        print("不能转生：" .. info.reason)
    end
    
    -- 显示最大转生次数
    print("最大转生次数: " .. info.maxRebirthCount)
end
```

问题排查：
==========
如果转生界面显示"未能获取等级信息"，可能的原因：

1. ❌ PlayerState为nil
   - 检查：UGCGameSystem.GetLocalPlayerState()是否返回有效对象

2. ❌ GetRebirthInfo函数不存在
   - 检查：playerState.GetRebirthInfo是否存在
   - 解决：确保UGCPlayerState.lua包含GetRebirthInfo函数

3. ❌ GameData未初始化
   - 检查：playerState.GameData是否存在
   - 解决：确保DataInit()已被调用

4. ❌ 复制属性未同步
   - 检查：UGCPlayerRebirthCount是否有值
   - 解决：等待服务器同步或调用RequestServerInit()
